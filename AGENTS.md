# AGENTS

本文件定义本仓库的系统层 agent 约束，适用于整个仓库。

## 语言要求

- 仓库内的文档、说明、注释、设计文案、页面展示文案，默认尽量使用中文。
- 新增 README、架构说明、服务拆分说明、部署说明、运维说明时，优先输出中文版。
- 如果引用外部英文术语，第一次出现时可保留英文原词，但应尽量配套中文说明。

## 代码与契约要求

- 代码中的目录名、模块名、接口路径、环境变量名、配置键名、结构体字段名、JSON 字段名，默认保持英文，避免影响服务契约稳定性。
- 不要为了中文化而随意改动已经对外暴露的 API 路径或字段名。
- 如需对英文接口做解释，应通过中文注释、中文文档或中文示例补充，而不是直接重命名接口。

## 注释要求

- 非自解释的关键代码块应补充简洁中文注释。
- 注释应解释设计意图、边界条件或服务职责，不要写低价值的逐行翻译式注释。

## Shell 约束

- 执行 Bash 命令时使用 Unix 风格重定向：`2>/dev/null` 而非 Windows 的 `2>nul`。
- `2>nul` 在 bash 环境下会创建字面量文件 `nul`，污染仓库。
- 判断文件/目录是否存在用 `test -f` / `test -d` 或专用工具（Glob/Grep），避免 `|| echo` 兜底。

## 交互约定

- 用户说"push 一下"时，自动检查改动、生成中文 commit message 并展示给用户确认，确认后直接 add + commit + push。
- 不要直接跳过确认步骤执行 push。

## 输出要求

- 面向研发、运维、产品或测试同学的仓库内说明材料，优先使用中文。
- 如果用户没有特别要求英文版，不要默认输出英文文档。
