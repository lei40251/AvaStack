AvaStack 学习路线
总览 先跑起来 Python 前端 语言学习
面向零基础 / 转行同学

AvaStack 实操型学习路线图

这份路线图的目标不是让你一下子学完所有技术，而是先把项目跑起来、看懂骨架、敢改小功能，再逐步具备继续推进这个仓库的能力。

  • 你可以先不会语言，也能先把项目跑起来。
  • 你不用先学完整门语言，再回来看项目。
  • 最好的学习方式是：看一点教程，立刻回仓库验证。

这份路线能帮你做到什么

把当前骨架项目跑起来，看懂服务职责，追一条请求链路，修改 stub 返回值，完成简单前端展示改动。

暂时不要给自己的目标

不是“立刻独立做完整数字人产品”，而是“先搞懂骨架、跑通流程、能做小改动”。

最重要的学习节奏

每次只学一个很小主题，最多 20 到 40 分钟，然后马上回项目里找对应代码。

项目技术全景

先知道项目里分别是谁在负责什么，再决定从哪里开始学。

你会碰到的主要技术

  • Python + FastAPI： 四个模型服务的接口骨架。
  • Go： 编排层，负责控制面、会话和健康聚合。
  • TypeScript + Vite： 管理后台页面。
  • Docker Compose： 本地把一组服务一起拉起来。
  • LiveKit / SRS： 后续实时音视频和分发能力。

建议的学习顺序

  1. 先把项目跑起来。
  2. 再看架构、服务边界和共享契约。
  3. 先读最简单的 Python 服务。
  4. 再看前端。
  5. 最后再进入 Go、LiveKit 和真实模型。

阶段 0：先把项目跑起来

很多小白卡住，不是因为不会写代码，而是因为项目从来没真正“活起来”。

0

目标：先看到系统在运行

先跑，不先啃语法

这一阶段只解决一件事：你能不能把项目拉起来，并知道它现在大概有哪几个入口。

README start.ps1 compose.yaml HTTP 接口验证

先看哪里

  • README.md
  • Docker Get started
  • Compose Quickstart

只看什么

  • 什么是镜像
  • 什么是容器
  • 什么是端口映射
  • 一个 compose.yaml 可以起多个服务

看完立刻做什么

  • 打开 compose.yaml
  • 找出哪个服务对宿主机暴露 58080，容器内实际监听仍是 8080
  • 找出哪个服务对应管理台
  • 运行 ./start.ps1
实践任务：启动完成后，访问 http://localhost:58080/healthz、/v1/info、/v1/sessions，再执行一次 POST /v1/sessions。

这一阶段应该做

  • 先确认项目能启动。
  • 先访问接口，看返回是什么。
  • 先知道系统有哪些服务。

这一阶段不要做

  • 不要现在去学写 Dockerfile。
  • 不要现在去背 Compose 全部语法。
  • 不要现在碰 Kubernetes。

阶段 1：看懂架构和服务边界

先知道“谁负责什么”，后面读代码就不会一直迷路。

1

目标：知道项目为什么这样拆

理解骨架，不求细节

当前仓库是服务化骨架，不是完整产品。先把这个定位认清，学习会轻松很多。

先读这 3 份文档

  • docs/architecture.md
  • docs/service-decomposition.md
  • shared/contracts/README.md
实践任务：用自己的话回答两个问题：为什么以后把 LLM 后端从 stub 换成 vLLM，不需要重写整个项目？为什么 orchestrator 不直接写死 ASR、TTS、LLM、Avatar 的全部逻辑？

阶段 2：先学最容易上手的 Python 服务

为了看懂项目，只补最少的 Python。不要等“学完 Python”才回来。

2

目标：看懂一个 FastAPI 服务长什么样

先读、先改、先验证

建议先看 model-asr-python、model-tts-python，再看 model-llm-python，最后看 model-avatar-python。

第一天这一轮

  • 看 Python 官方教程
  • 只看变量、if、函数、列表、字典
  • 回到 model-asr-python/app/main.py
  • 只找 app = FastAPI(...)、@app.get("/healthz")、return {...}

第二天这一轮

  • 看 FastAPI 官方教程
  • 只看 First Steps、Path Operation、Request Body
  • 回到 model-llm-python/app/main.py
  • 只找 ChatRequest、@app.post("/v1/chat")、payload.message

第三天这一轮

  • 如果还不理解 BaseModel，再看 Pydantic Get Started
  • 把 stub response 改成你自己的文本
  • 重启服务
  • 验证接口返回有没有变化
实践任务：当前仓库里不是 /v1/chat/completions，而是 POST /v1/chat。你要能自己在代码里找到它，并说清楚它收什么、返回什么。

阶段 3：理解共享契约和会话模型

服务不是随便返回 JSON 的。你后面改 Go、改 Python、改前端，都离不开这套结构。

3

目标：知道接口外层结构为什么要统一

抓住 request_id / session_id / status / data / meta

重点看

  • 通用响应包裹
  • 错误响应约定
  • Session Schema
  • 会话创建与更新请求
实践任务：先创建一个会话，再调 GET /v1/sessions，然后对照契约文档，把返回字段一一对应上。

阶段 4：进入 Go 编排层

这里是项目的“大脑”，但你不需要先把 Go 学完整，再来看它。

4

目标：能追一条控制面请求链路

从接口路径追到处理函数

先看哪里

  • Get started with Go
  • A Tour of Go
  • router.go

只学最小知识

  • func
  • struct
  • map
  • if err != nil
  • func (r *Router)

回仓库先看什么

  • cmd/api/main.go
  • 路由注册
  • SessionStore
实践任务：找到 POST /v1/sessions 对应的函数，画出它从路由到会话创建再到响应返回的最短调用链。

阶段 5：理解管理后台前端

前端是最容易做出可见改动的地方，很适合建立“我真的能改动项目”的感觉。

5

目标：看懂页面怎么拿数据、怎么显示数据

不先学框架，先学原生 TS

第一天这一轮

  • 看 TypeScript Handbook
  • 只看 type、对象类型、函数
  • 回到 main.ts
  • 只找 ServiceHealth、SessionItem、fetchJSON()

第二天这一轮

  • 看 MDN Fetch API
  • 回到 bootstrap()
  • 回答：它先请求了哪个接口？失败时页面怎么提示？

第三天这一轮

  • 如果 DOM 很陌生，再看 MDN JS 学习路径
  • 回来看 renderHealth() 和 renderSessions()
  • 把“当前还没有会话”改成你自己的提示语
实践任务：给会话列表多展示一个字段，或者给服务健康状态改一个更醒目的提示文案。

语言学习细化版

不是让你把整门语言学完，而是先学到“能读、能改、能验证”。

Python：第一优先级

  • 先学变量、函数、字典、列表、类。
  • 把每个概念都回仓库里找一次。
  • 你现在最重要的目标，不是“精通 Python”，而是敢改一个 FastAPI 服务的返回值。

最省力的资料顺序

  • Python 官方教程（中文）
  • FastAPI 官方教程（中文）
  • Pydantic Get Started
  • venv 官方文档

TypeScript：第二优先级

  • 先看类型定义、函数、fetch、DOM 查询。
  • 始终围绕 3 个问题：这个类型约束什么？这个请求打哪个接口？这段模板显示到哪？
  • 最先要做到的是改一个页面展示细节。

最省力的资料顺序

  • TypeScript Handbook
  • MDN JavaScript 学习路径
  • MDN Fetch API
  • Vite 官方中文文档

Go：第三优先级

  • 先学 func、struct、map、if err != nil。
  • 把 Go 当成“为了解一条请求链路才学”，不是“先学会 Go 才能看项目”。
  • 先追路由，再看会话存储，再看状态流转。

最省力的资料顺序

  • Get started with Go
  • A Tour of Go
  • Effective Go

完全零基础时的节奏建议

第 1 周

先跑项目，同时补一点 Python 基础，看最简单的 ASR / TTS 服务。

第 2 周

继续补 FastAPI，看 LLM 服务，改一次 stub 返回值。

第 3 周

开始补 TypeScript，看管理台页面，改一次文案或字段展示。

第 4 周

开始补 Go，追一条 POST /v1/sessions 的请求链路。

阶段 7：最后再碰 LiveKit、SRS 和真实模型

这些都更偏进阶，不要一开始就拿最重的主题吓自己。

现在只需要知道什么

  • infra/livekit/livekit.yaml 是 LiveKit 的基础配置。
  • infra/srs/srs.conf 是 SRS 的基础配置。
  • infra/vllm/README.md 说明未来 LLM 真实化方向。
